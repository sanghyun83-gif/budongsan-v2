# SALJIP_HOME_MINIMAL_SEARCH_FIRST_SPEC_2026-04-05

## 목적
- 랜딩(`/`)을 검색 퍼스트 UX로 단순화해 첫 진입 의사결정 시간을 줄인다.
- 핵심 플로우(`검색 → 맵 줌인 → 핀 클릭 → 상세`)를 랜딩에서 즉시 시작 가능하도록 고정한다.

## 최종 판정
- 현재 `/`는 정보/보조 블록 밀도가 높아 첫 진입 판단이 느리다.
- 목표 `/`는 **검색 입력 + 지도/결과** 중심의 미니멀 레이아웃이다.

## 적용 원칙
1. 첫 화면 행동은 1개만: `검색`
2. 정보 우선순위: 검색 > 결과/지도 > 보조 안내
3. 보조 문구는 1줄 원칙(출처/업데이트)
4. 내부 운영/개발 문구 사용자 노출 금지

## IA/레이아웃 (검색 퍼스트)

### 상단
- 로고(작게) + 검색창(가장 크게)
- 불필요 CTA/보조 버튼 비노출

### 본문
- 좌: 검색 결과 리스트
- 우: 지도
- 초기 상태에서 즉시 검색/탐색 가능

### 하단
- 정책 링크(About/Privacy/Terms)는 유지하되 시각 우선순위 하향
- 과도한 설명문/마케팅 문구 제거

## 제거/하향 대상
- 과다 설명 텍스트
- 부가 카드/추천 블록의 랜딩 우선 노출
- 검색과 무관한 안내 문구
- 개발자/운영 상태 문구

## 유지 대상
- 검색 입력
- 지도/결과 동시 탐색
- 출처/업데이트 최소 안내(1줄)

## 완료 조건 (Done)
- `/` 첫 화면에서 검색창과 지도/결과가 최우선으로 보임
- 첫 화면 텍스트 블록이 검색 행동을 방해하지 않음
- `검색 → 맵 줌인 → 핀 클릭 → 상세` 동선이 3클릭 내 성립
- lint/build 통과
