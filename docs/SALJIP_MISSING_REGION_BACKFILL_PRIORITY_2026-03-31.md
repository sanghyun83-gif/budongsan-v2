  - `notes/hub-search-db-audit-2026-03-31.json`
  - DB 직접 점검(전체 `complex` 8,258건)
- 우선순위 조건:
  - 사용자 검색 실패 체감이 큰 생활권
  - 브랜드/동 조합 검색 누락이 잦은 구간
  - 검색량이 높은 롱테일(동+브랜드) 우선

## 핵심 발견
- 전체 단지: `8,258`
- 정규화 self-match 실패군: `609` (정규화 취약군)
- 공백 제거 self-match 실패군: `92` (즉시 개선 가능군)
- 사용자 이슈 직접 확인:
  - `고덕아이파크` 매칭 0건
  - `고덕`/`상일` 법정동 매칭 0건
  - 즉, 이 케이스는 검색식보다 **DB 미적재** 문제

## Backfill 우선순위
1. P0 (즉시) — 강동 고덕/상일 생활권
- 대상: `고덕`, `상일` 키워드 매칭 단지군 전량
- 이유: 실제 사용자 질의가 이미 발생했고, 결과 0건으로 UX 손실이 큼
- 목표: `고덕아이파크`, `고덕센트럴...` 계열 최소 커버리지 확보

2. P1 (이번 주) — 서울 핵심 브랜드 동 조합
- 대상 예시:
  - `문정 래미안`, `잠실 아이파크`, `대치 자이`, `반포 더샵`, `상계 롯데캐슬`
- 이유: 브랜드+동 검색은 전환 의도가 높아 우선 투자 가치가 큼

3. P2 (다음 주) — 수도권 외곽 인기 신축 클러스터
- 대상:
  - 하남/동탄/판교/광교/검단 등 최근 거래량 상위 지역
- 이유: 허브 KPI/트래픽 확장과 직접 연결

## 실행 단위 (권장)
- 단위 1: `법정동` 기준 배치 backfill
- 단위 2: `브랜드 사전` 기준 보정 backfill
- 단위 3: 검색 로그 기준 “0건 키워드” 주간 반복 보강

## 완료 기준
- `고덕`/`상일` 키워드 검색 0건 해소
- 상위 브랜드 키워드(래미안/아이파크/자이/더샵/롯데캐슬) 0건 비율 50% 이상 감소
- 허브 검색 0건률 주간 감소 추세 확인
