## Phase 1 (D1~D3): 맵 UX 최소 안정화 (30% 트랙)
### Step 1-1. 검색 중심 흐름으로 고정
- 원칙: 검색 전 지도 과밀 노출 금지
- 동작:
  - 검색어 입력 후 결과 생성
  - 결과가 있으면 맵 자동 줌인
  - 핀 클릭 시 우측 패널/상세로 연결

### Step 1-2. 자동 리셋/자동 바운드 점프 제거
- 증상 개선:
  - 사용자가 줌/이동한 직후 기본맵 복귀 방지
  - 검색 중 화면이 임의로 튀는 현상 방지

### Step 1-3. 마커 밀도 제어
- 줌레벨 기반 마커 수 제한 또는 클러스터링
- 저줌에서 과밀 렌더링 방지

### Step 1-4. UI 단순화
- 제거/축소:
  - 개발중 인상 문구(`MVP`, `준비중`, 코드값 노출)
  - 중복 범위 토글
- 유지:
  - 검색창 + 정렬 + 최소 필터 + 결과패널

완료 조건:
- `npm run lint` 통과
- `npm run build` 통과
- 수동 스모크(검색→줌인→핀클릭→상세) 정상

## Phase 2 (D2~D10): 텍스트 SEO 대량 페이지 엔진 구축 (70% 트랙)
### Step 2-1. 템플릿 고정
- 상세 상단 요약문 템플릿 통일
- 메타 템플릿 통일:
  - `title`, `description`, `og:title`, `og:description`
- `data-nosnippet` 적용 범위 확정(금융/보조 블록)

### Step 2-2. 발행 단위 정의
- 우선: 아파트+매매 중심
- URL 유형:
  - 단지 상세
  - 지역/동 허브
  - 브랜드 허브(래미안/자이/힐스테이트 등)

### Step 2-3. 발행 우선순위 큐 생성
- 소스:
  - GSC 노출 쿼리
  - 네이버/구글 샘플링 문서
- 우선순위 규칙:
  - 노출↑ 클릭↓ 키워드
  - 거래량/검색수 높은 지역/브랜드
- 산출물:
  - `docs/SALJIP_TEXT_PAGE_PRIORITY_QUEUE_2026-04-03.md`

### Step 2-4. 내부링크 강화
- 상세당 8~12개 관련 링크
- 지역/브랜드/유사단지 링크 연결

완료 조건:
- 신규/개선 URL 묶음 반영
- 내부링크 누락/깨짐 없음

## 맵 단일 플로우 운영 원칙 (2026-04-03)

### 최종 채택 패턴
- 1인 운영 기준으로 홈 탐색은 `검색 → 맵 자동 줌인(1회) → 핀 클릭 → 상세 진입` 단일 플로우로 고정한다.

### 왜 이 패턴을 채택하는가
- 개발/운영 비용이 가장 낮다.
- 사용자 학습 비용이 낮아 초기 이탈을 줄이기 쉽다.
- 현재 목표(빠른 유입 + 완성도 확보)에 가장 직접적으로 기여한다.

### 필수 가드레일 (반드시 준수)
1. 검색 시 자동 줌인은 `1회`만 수행(반복 리셋 금지)
2. 마커 과밀 제어 적용(저줌 마커 제한 또는 클러스터)
3. 핀 클릭 후 상세 진입 경로 단순화(우측 패널 최소 정보)
