# SALJIP Hub Manual Smoke Checklist (2026-04-04)

## 목적
- 최종 채택 패턴(`검색 → 맵 줌인 → 핀 클릭 → 상세`)이 실제 UI에서 안정적으로 동작하는지 확인한다.
- 자동 QA(`lint/build`) 통과 이후, 사용자 체감 동선/가시성만 수동으로 검증한다.

## 사전조건
- Node/의존성 설치 완료
- `.env.local` 유효
- 로컬 서버 실행: `npm run dev`
- 접속 URL: `http://localhost:3000`

## 완료 기준 (PASS 조건)
1. 검색 후 자동 줌인은 1회만 수행되고, 이후 사용자 수동 이동을 강제로 되돌리지 않는다.
2. 저줌 과밀 구간에서도 핀 클릭이 가능하다(마커 과밀 제어 체감).
3. 핀 클릭 후 상세 진입까지 2스텝 이내다.
4. 상단 UI에 개발중 인상 문구(`MVP`, `준비중`)가 노출되지 않는다.

## 테스트 케이스

### TC-01 검색 → 자동 줌인 1회
- 단계
  1) 홈 접속
  2) 검색어 `래미안` 입력 후 검색
  3) 맵 자동 이동/줌 확인
  4) 맵을 임의로 드래그/줌 변경
  5) 2~3초 대기
- 기대결과
  - 검색 직후 1회만 자동 줌인 발생
  - 사용자 수동 이동 후 자동으로 원위치 리셋되지 않음

### TC-02 저줌 과밀 제어
- 단계
  1) 맵을 저줌(넓은 범위)으로 이동
  2) 핀 밀집 지역에서 클릭 시도
- 기대결과
  - 화면이 핀으로 과도하게 막히지 않음
  - 임의 핀 클릭 가능

### TC-03 핀 클릭 → 상세 진입
- 단계
  1) 보이는 핀 1개 클릭
- 기대결과
  - 단지 상세(`/complexes/:id`)로 이동
  - 총 동선 2스텝 이내 유지

### TC-04 우측 패널 경유 상세(보조 동선)
- 단계
  1) 검색 결과 목록에서 단지 1개 선택
  2) `선택 단지` 탭에서 `단지 상세 보기` 클릭
- 기대결과
  - 상세 페이지 정상 이동
  - 선택 단지 정보(최근 거래가/거래일/입주연차/세대수) 노출

### TC-05 상단 카피/노출 점검
- 단계
  1) 홈 상단 영역 시각 점검
- 기대결과
  - `MVP`, `준비중` 등 개발중 인상 문구 미노출
  - 거래 유형/매물 유형/검색 중심 구조 유지
