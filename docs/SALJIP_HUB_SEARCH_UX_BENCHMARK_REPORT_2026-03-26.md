# SALJIP_HUB_SEARCH_UX_BENCHMARK_REPORT_2026-03-26

## 1) 보고서 목적
- 문제 정의: 현재 허브(`http://localhost:3000`)는 검색 시 지도 바운드 내 결과만 반환되어, 사용자가 원하는 단지를 놓치는 UX가 발생함.
- 목표: Top-tier 부동산 서비스 패턴을 기준으로 살집 허브의 검색/지도 결합 구조를 재설계.
- 작성일: 2026-03-26 (KST)

## 2) 현재 허브 동작 진단 (코드 근거)
- URL 동기화 시 바운드(`sw_lat/sw_lng/ne_lat/ne_lng`)를 항상 포함함.
- 검색 요청(`/api/search`)에도 바운드를 항상 포함해 전송함.
- 지도 이동 후 디바운스(450ms)로 자동 재검색이 실행됨.
- 결과적으로 “키워드 검색 = 현재 지도 내 검색”으로 강결합되어, 전국 단지 탐색성이 낮아짐.

## 3) Top-tier 전수조사 요약

### 조사 대상
- 직방
- 다방
- 호갱노노

### 공통 패턴 (교집합)
- 검색 필터가 풍부하고, 거래 유형(매매/전세/월세)을 명시적으로 분리.
- 지도 기반 탐색은 강하게 제공하지만, 지도 이동 즉시 강제 재검색보다 “사용자 의도 기반 갱신” 패턴이 많음.
- 사용자가 “전국/지역/지도 범위”를 인지할 수 있는 상태 신호를 제공.

### 서비스별 관찰
- 직방: 검색 필터를 전면 제공하고 지도 위 시세/트렌드 정보를 결합함.
- 다방: 지도 화면에서 “살고 싶은 위치를 그려 조건 매물 확인”을 명시함(사용자 의도형 영역 검색).
- 호갱노노: 검색/필터 축이 매우 다양하며(매매/전세/월세 포함), 지도 기반 탐색과 상세 정보 축이 명확함.
- 아실: 매매/전세 비교, 지도 기반 지역 개발 정보 확인 등 “비교+지도” 흐름을 강조.
- KB부동산: 시세/실거래가/분양/예측시세를 허브 메시지로 통합 노출.

## 4) 핵심 갭 분석
- 현재 살집 허브는 검색 범위 제어 UI가 없어 사용자가 “지도 내 검색 강제”를 인지/제어하기 어려움.
- 지도 이동 자동 재검색이 잦아 리스트가 흔들리고, 검색 의도(키워드)보다 지도 상태가 우선됨.
- Top-tier 대비 부족한 점은 “검색 범위 명시 + 갱신 타이밍 제어 + 거래유형 축 확장”임.

## 5) 허브 개편안 (권장)

### A. 검색 범위 분리 (가장 우선)
- 검색 스코프 토글 추가: `전국 검색` / `지도 내 검색`
- 기본값: `전국 검색`
- 지도 바운드가 강제되지 않도록 검색 의도 우선.

### B. 지도 갱신 트리거 변경
- 현행: 지도 pan/zoom 후 자동 재검색
- 변경: 지도 이동 시 상단 고정 버튼 노출 `이 지도에서 검색`
- 기대효과: 불필요한 재요청 감소, 사용자 제어감 상승, 리스트 안정성 개선.

### C. 리스트/지도 역할 분리
- 리스트 API(`/api/search`): 스코프에 따라 전역 또는 바운드 적용.
- 지도 API(`/api/map/complexes`): 바운드 유지(현행 유지).
- 리스트가 지도 밖 결과를 포함할 때 안내 배지 표시: `지도 밖 결과 포함`.

### D. 결과 상태 가시화
- 결과 헤더에 범위 문구 명시:
- `검색 범위: 전국` 또는 `검색 범위: 현재 지도`
- 카운트 분리 표시:
- `리스트 N건 · 지도 M건`

### E. 거래유형 축 확장 (후속)
- 허브 필터에 거래유형 탭 추가: `매매 / 전세 / 월세 / 전체`
- 현재 매매 중심 구조에서 top-tier 수준의 기본 탐색 축으로 확장.

## 6) API/프론트 변경 설계

### API
- `/api/search` 파라미터 추가:
- `scope=global|map` (default: `global`)
- `scope=global`이면 bbox 무시, `scope=map`이면 bbox 필수 검증.
- `/api/hub/recommendations`도 동일 스코프 파라미터 적용 권장.

### 프론트
- `components/Explorer.tsx`
- 상태 추가: `searchScope`, `hasPendingMapChange`
- 지도 이동 시 자동 fetch 제거.
- `이 지도에서 검색` 버튼 클릭 시에만 `scope=map` 요청.
- 수동 검색(Enter/버튼)은 기본 `scope=global`.

## 10) 리스크/주의사항
- 글로벌 검색 결과가 많아질 수 있으므로 정렬/페이지네이션 최적화 필요.
- 기존 사용자(지도 중심)에겐 자동갱신 제거가 다르게 느껴질 수 있어 버튼 가시성 강화 필요.
- 추천/스냅샷/생활인프라 블록은 스코프 기준을 명확히 통일해야 혼란이 없음.

## 11) 최종 제안 (결론)
- 현재 허브는 “지도 바운드 강결합 검색”이 핵심 UX 병목.
- top-tier 교집합을 반영하면, 우선순위는 아래 순서가 가장 효율적임:
- 1) 검색 범위 분리(`전국` 기본)
- 2) 지도 수동 갱신 버튼(`이 지도에서 검색`)
- 3) 결과 상태 가시화(범위/카운트)
- 4) 거래유형 확장(매매/전세/월세)
