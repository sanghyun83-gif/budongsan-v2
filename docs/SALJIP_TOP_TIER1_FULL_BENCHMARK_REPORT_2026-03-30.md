# SALJIP_TOP_TIER1_FULL_BENCHMARK_REPORT_2026-03-30

## 목적
- `saljip.kr` 허브(`http://localhost:3000/`)를 국내 상위 부동산 서비스 패턴에 맞춰 재설계하기 위한 전수조사 리포트.
- 요청사항 기준:
  - 허브부터 상위 서비스 공통 기능은 최대한 흡수
  - 상위 서비스에 거의 없는 기능/노출은 축소 또는 제거
  - “모든 요소 결합” 방향으로 설계하되, 브랜드/문구/자산의 직접 복제는 제외(법적 리스크 방지).

## 조사 범위 (Top-tier)
- 호갱노노
- 아실
- 직방
- 다방
- KB부동산
- 네이버페이 부동산

## 조사 방법
- 공식 웹/앱 소개 페이지, 공식 서비스 페이지, 공식 뉴스룸/운영 페이지 기준으로 기능 수집.
- 광고/블로그 2차 가공 페이지는 원칙적으로 배제.
- 동적 렌더링/robots 제한으로 본문 수집이 제한된 경우, 공식 앱 설명(Play), 공식 서브페이지, 공식 보도자료로 보완.

## 1) 플랫폼별 핵심 기능 정리

### A. 호갱노노
- 지도+검색 기반 아파트 탐색.
- 고급 필터 폭이 넓음: 매매/유형/평형/가격/세대수/입주년차/용적률/건폐율/전세가율/갭가격/월세수익률/난방방식 등.
- 상단 핵심 탭이 명확함: `분양`, `이야기`, `경매`, `뉴스`, `오늘`.
- 커뮤니티/입주민 인증/채팅방 흐름이 붙어 있음.
- 분양/경매 섹션이 별도 탐색 표면으로 강함.

### B. 아실
- 지도 중심 데이터 탐색 + 지표형 분석(공급/인구/가격/심리/거래).
- 실거래/매물/평당가/갭/전세가율/거래회전율 등 “투자·분석형” 지표 강세.
- 분양/입주물량/학군/단지비교/시장분석까지 묶여 있음.
- 매매·전세 동시비교, A/B 단지 비교 같은 비교 기능이 강점.

### C. 직방
- 멀티 카테고리(아파트/분양/원룸/오피스텔/빌라/상가) 통합 탐색.
- 지도 우선 UX + 단지 선택 시 우측 패널(단지 핵심정보/가격/그래프/거래) 노출 패턴.
- AI중개사(대화형 추천), 채팅 문의, 3D 단지투어 등 탐색→전환 연결이 강함.
- 중개사·광고 생태계(매물 노출 채널) 구조가 명확함.

### D. 다방
- 멀티 카테고리(주택/빌라/오피스텔/아파트) 통합.
- 커뮤니티/동네 기반 콘텐츠와 매물 탐색 결합.
- 단지 랭킹, 평당가/갭 등 단지 정보 확장 제공.

### E. KB부동산
- 시세 엔진 중심: `KB시세 + 실거래가 + 매물가 + 공시가 + AI예측시세`.
- 단지 페이지 정보량 큼: 시세/실거래/타입/기본정보/학군/커뮤니티(단지글)/매매전월세.
- 지도 레이어도 풍부(준공년도/실거래가/세대수/전세가율/학군 등).
- 금융/대출 CTA와의 연결이 빠름.

### F. 네이버페이 부동산
- 대형 트래픽 관문 구조: `매물/분양/뉴스/커뮤니티/MY/우리집`.
- 거래유형 필터가 명확(매매/전세/월세/단기).
- 단지 페이지 탭: 시세/실거래, 매물, 단지정보, 이야기 + 기간별(1/3/5/7년) 흐름.
- 지역 선택 UX와 지도 핀/필터 안내가 체계적.

## 3) 현재 Saljip 허브 대비 Gap

## 이미 좋은 축
- 지도 + 검색결과 2단 구성
- 검색 스코프(`전국`/`현재 지도`)와 `이 지도에서 검색` 버튼
- KPI/트렌드/스냅샷/생활인프라/조건기반추천 등 확장 모듈
- 데이터 출처/업데이트 노출

## 부족한 축 (Top-tier 대비)
- 거래유형(매매/전세/월세) 1차 탭 부재
- 부동산 유형(아파트/오피스텔/빌라 등) 1차 탭 부재
- 필터가 얕음(입주연차/세대수/전세가율/갭/학군/교통 등 부족)
- 지도 선택 단지의 우측 고정 패널 흐름 약함
- 단지 비교(A/B) 부재
- 매물(실제 listing) 섹션 연동 약함
- 커뮤니티/리뷰/이야기 계층 부재

## 5) 허브 재설계 제안 (No-compromise 패턴 결합안)

## 허브 정보구조 (Top fold)
1. 상단 검색바: `키워드` + `지역` + `거래유형(매매/전세/월세)` + `유형(아파트/오피스텔/빌라...)`
2. 필터 드로어: 가격/면적/입주연차/세대수/학군/교통/전세가율/갭
3. 본문: `좌측 지도(대)` + `우측 결과 패널(고정)`
4. 지도 인터랙션: 자동재검색 OFF(이미 적용됨), `이 지도에서 검색` 버튼 ON

## 단지 클릭 UX
- 지도 핀 또는 리스트 클릭 시 우측 패널 탭:
  - `검색결과` / `선택단지` / `추천`
- 패널 상단 핵심 정보:
  - 최근 거래가 / 최근 거래일 / 최근 3개월 거래량
- CTA는 현재 단계에서 최소화:
  - `단지 상세 보기` 중심
  - 관심등록/알림은 로컬 토글 수준까지만 유지
  - 문의 채널은 보류(유입 검증 후 재도입)

## 허브 하단 확장 영역
- 트렌드/KPI/급상승 키워드/추천/생활인프라는 하단 섹션 유지
- 단, “탐색 핵심 흐름(검색-지도-결과)”보다 위로 올리지 않음

## 6) 실행 우선순위 (2026-04-04 재정의)

### Phase A (지금 즉시 유지할 코어)
1. 단일 탐색 플로우 안정화 (`검색 → 줌인 → 핀 클릭 → 상세`)
2. 거래유형/유형은 `매매 + 아파트` 단일 축 우선 유지
3. 우측 패널은 `검색결과/선택단지/추천` 최소 탭 유지
4. 필터는 실제 동작 항목만 노출(placeholder 필터 제거)

### Phase B (조건부 보류)
1. 문의 채널/문의 CTA 운영
2. 채팅형 문의 흐름
3. 매물 외부 연동 강화
4. 전세/월세, 오피스텔/빌라/원룸 활성화

### Phase C (트래픽 검증 후)
1. A/B 단지 비교
2. 랭킹/시장분석 고도화
3. 커뮤니티(이야기/리뷰)

## 8) 결론
- 현재 Saljip는 데이터/인사이트 방향은 강점이 있으나, 탐색 기본 UX는 Top-tier 대비 얕음.

## UX 확정안 (2026-03-30)

### 확정 원칙
- 홈 상단은 `검색창 + 1차 토글(거래유형/유형) + 정렬 + 검색 버튼`만 우선 배치한다.
- 운영 정보/보조 정보(출처, 업데이트, 코드성 옵션)는 상단에서 분리하고 필요 시 2차 영역으로 이동한다.
- 데이터 신뢰 신호(출처, 업데이트 시각, 기준)는 결과 영역 하단에 명시한다.

### 상단 카피/노출 정리
- 기본 화면에서 `준비중`, `MVP`, 코드성 상태 문구는 제거한다.
- 아래 항목은 기본 화면 노출을 최소화하고 고급/보조 영역으로 이동한다.
  - `정확 좌표만 보기(exact)`
  - `지역코드 5자리` 직접 입력

### 검색 UX 구성
- 기본 동선:
  - 검색어 입력
  - 거래유형 탭(현재 `매매` 활성)
  - 유형 탭(현재 `아파트` 활성)
  - 정렬
  - 검색 버튼
- 비활성 옵션은 문구로 노출하지 말고, disabled 스타일로만 일관 처리한다.

### 결과/빈결과 UX
- 0건 상태는 단순 명확하게 유지:
  - `조건에 맞는 단지가 없습니다.`
  - `조건 초기화` 버튼
- 우선순위는 `지도 + 우측 결과 패널(검색결과/선택단지/추천)`에 둔다.
- KPI/트렌드/스냅샷/추천 확장 섹션은 하단 유지.

### 작업 우선순위
1. 상단 UI 정리(문구/위계)
2. 고급 입력(지역코드/exact) 기본 화면 분리
3. 0건 상태 단순화
4. 지도+우측패널 중심 레이아웃 안정화

## 카피/문구 정리안 (2026-03-31)

### 문제 진단
- 기존 홈 카피에 `MVP`, `준비중`, 내부 상태를 연상시키는 표현이 혼재되어 신뢰감을 떨어뜨림.
- Top-tier 서비스 1화면은 사용자 행동 중심 문구만 남기고 내부 단계 표현을 거의 노출하지 않음.
- 첫 방문 사용자는 “무엇을 입력하고 검색하면 되는지”만 빠르게 이해하면 충분함.

### 확정 카피 원칙
1. 내부 단계 문구 제거: `MVP`, `준비중` 등은 홈 기본 노출에서 제외
2. 용어 단순화: 거래유형/유형은 버튼형으로 직관화
3. 1차 행동 우선: 검색창 + 1차 토글 + 정렬 + 검색 버튼 고정
4. 운영/품질 정보 후순위: 출처/업데이트/보조옵션은 결과 하단으로 이동

### 홈 카피 교체안
- 헤더
  - 기존: `검색+추천+지도 탐색 MVP` 류 문구
  - 변경: `서울·수도권 아파트 실거래가·시세를 지도에서 검색하세요`

- 거래유형 영역
  - 기존: 긴 설명형 문구
  - 변경: `거래 유형` + 탭 버튼(매매/전세/월세)

- 유형 영역
  - 기존: 긴 설명형 문구
  - 변경: `매물 유형` + 탭 버튼(아파트/오피스텔/빌라/원룸)

### 보조 옵션/이동 정리
- 홈 기본 화면에서는 다음 문구/입력을 제거 또는 축소:
  - `(전세/월세/오피스텔/빌라/원룸 준비중)`
  - `지역코드: ...`
  - `정확 좌표만 보기`
- 필요 시 필터 드로어/고급 설정으로 이동.

### 0건 상태 카피
- 본문: `조건에 맞는 단지가 없습니다.`
- 버튼: `조건 초기화`
- 원칙: 빈 상태에서 선택지를 과도하게 늘리지 않고 재검색 액션만 제공.

### 실행 우선순위
1. 상단 카피에서 `MVP/준비중` 제거
2. 거래유형/유형 버튼 라벨 정리
3. 보조 문구/코드성 입력 이동

## 최종 채택 패턴: 맵 단일 탐색 플로우 (2026-04-03)

### 의사결정
- Top-tier 기능을 전부 동시 구현하지 않고, 현재 단계에서는 `직관적인 단일 맵 탐색 플로우`를 우선 채택한다.
- 기준 플로우: `검색어 입력 → 해당 지역/단지로 맵 줌인 → 핀 클릭 → 상세 진입`.

### 채택 이유 (1인 운영 기준)
- 범용 기능을 넓게 깔기보다 핵심 플로우 완성도가 유입/체류에 더 큰 영향.
- 개발/운영/디버깅 비용을 낮추고 배포 속도를 높일 수 있음.
- 학습 비용이 낮아 첫 방문 사용자의 탐색 실패를 줄일 수 있음.

### 구현 원칙
1. 자동 줌인 1회만 허용 (검색 후 반복 리셋 금지)
2. 저줌 과밀 마커 제어 (제한/클러스터)
3. 핀 클릭 이후 동선 단순화 (우측 패널 최소 정보 또는 바로 상세)
4. 상단 UI 문구/토글 과잉 제거 (개발중 인상 문구 비노출)

### 완료 기준
- 검색 후 맵 이동/줌이 안정적으로 고정되고, 사용자 수동 이동을 침해하지 않음
- 과밀 구간에서도 클릭 가능한 수준의 시인성 유지
- 핀 클릭 → 상세 진입까지 2스텝 이내 유지
