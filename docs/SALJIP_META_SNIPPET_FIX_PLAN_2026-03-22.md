# SALJIP Meta Snippet Fix Plan (2026-03-22)

## 목적
- 경쟁사 문구를 복붙하지 않고, `saljip.kr` 고유 문구로 검색 스니펫 품질을 안정화한다.
- 검색 결과에서 금융 계산 텍스트가 우선 노출되는 현상을 줄이고, 단지 핵심 의도(실거래가/시세/매매·전세·월세/최근거래)를 먼저 보이게 만든다.

## 적용 원칙
- 문구 모방 금지: 구조와 정보 우선순위만 참고한다.
- 메타 태그만 믿지 않는다: 실제 스니펫은 본문/OG/기타 요소에서 자동 추출될 수 있다.
- 변경 후 반드시 재수집 요청 + 모니터링까지 완료한다.

---

## 핵심 수정 포인트 (복붙 모방 아님)

### 1) 상세 상단 요약문을 스니펫용으로 강화
목표: 검색엔진이 본문에서 스니펫을 뽑아도 핵심 정보가 먼저 노출되게 만든다.

- 상단 첫 요약문(본문 초반)에 아래 요소를 모두 포함:
- `실거래가`
- `시세`
- `매매·전세·월세`
- `최근 거래`

- 권장 문장 템플릿(예시):
```text
{법정동} {단지명} 아파트의 매매·전세·월세 실거래가와 시세를 확인하세요. 최근 거래일, 최근 거래가, 평형별 가격, 최근 3개월 거래량을 제공합니다.
```

- 보조 문장(데이터 신뢰성):
```text
출처: 국토교통부 실거래가 공개데이터 · 기준: API 조회 시점
```

- 구현 체크:
- 상세 페이지 상단에서 제목 바로 아래에 1~2문장으로 배치
- 숫자 나열형 문구 대신 검색 의도 중심 문장 사용
- 첫 화면(스크롤 전)에 요약문이 보이도록 유지

---

### 2) 금융 계산 블록은 페이지 하단으로 이동
목표: 금융 계산 문구가 스니펫 후보의 최상단 텍스트로 잡히는 문제를 완화한다.

- 레이아웃 권장 순서:
1. 단지 헤더(이름/지역/신뢰칩)
2. 스니펫용 요약문
3. 핵심 지표(최근 거래가/최근 거래일/최근 3개월 거래)
4. 거래 내역/생활 인프라 등 핵심 부동산 정보
5. 금융 계산 블록(하단)

- 구현 체크:
- `FinanceEstimateCard`는 상세 페이지 하단 섹션으로 이동
- 상단/중단에 남아 있는 금융 계산 문구 제거

---

### 3) Google용으로 금융 블록에 `data-nosnippet` 적용 검토
목표: Google이 금융 계산 텍스트를 스니펫에 과도하게 사용하지 않도록 제어한다.

- 적용 대상(권장):
- `예상 대출원금`, `월 상환액 예상`, `총이자(예상)` 등 계산 결과 라인
- CTA 주변 안내 문구(검색 의도와 직접 연관이 낮은 부분)

- 적용 예시(React):
```tsx
<section id="finance-estimator" data-nosnippet>
  ...
</section>
```

- 주의:
- `data-nosnippet`은 Google 중심 제어이며, 네이버 반영은 보장되지 않음
- 페이지 핵심 가치 문구(실거래가/시세/거래 정보)에는 적용하지 않음

---

### 4) `title` / `description` / `og:title` / `og:description` 메시지 일치
목표: 채널별(검색/공유) 메시지 불일치를 줄이고, 검색엔진 해석 혼선을 줄인다.

- 단일 메시지 규칙:
- `title`: 단지명 + 실거래가·시세 + 브랜드
- `description`: 실거래가/시세/매매·전세·월세/최근거래 요약
- `og:title`, `og:description`: 위와 동일한 의미로 통일

- 권장 예시:
```text
title: {단지명} 실거래가·시세 (매매·전세·월세) | 살집
description: {법정동} {단지명} 아파트의 매매·전세·월세 실거래가와 시세, 최근 거래일, 평형별 가격, 최근 3개월 거래량을 확인하세요. 출처: 국토교통부 실거래가 공개데이터.
```

- 구현 체크:
- 상세 `generateMetadata`에서 `description` 1원본을 만들고 OG/Twitter에 재사용
- canonical URL 정상 유지
- 페이지 본문 상단 요약문과 메시지 톤 일치

---

### 5) 변경 후 GSC/네이버 URL검사로 재수집 요청
목표: 반영 지연을 줄이고, 스니펫 개선 결과를 빠르게 확인한다.

#### Google Search Console
1. URL 검사에 상세 URL 입력 (`/complexes/{id}`)
2. 라이브 테스트 실행
3. 색인 생성 요청 클릭
4. 대표 샘플 URL 5~10개 반복 요청
5. 3~7일 후 검색 결과 스니펫 확인

#### Naver Search Advisor
1. URL 검사(또는 웹페이지 수집 요청)에서 상세 URL 입력
2. 수집 요청 실행
3. 요청 이력/수집 상태 확인
4. 3~7일 후 네이버 웹검색 스니펫 재확인

#### 검증 기준
- 기존: 금융 계산 텍스트가 스니펫 첫 줄 우선 노출
- 목표: 실거래가/시세/매매·전세·월세/최근거래 중심 스니펫 노출

### 운영 원칙 (중요)
- 허브 메타는 짧고 안정적으로 유지한다.
- 금융 계산 문구/동적 숫자는 상세 페이지 본문 하단으로만 노출한다.
- 허브 첫 화면 상단 문장(본문)도 메타와 톤 일치 상태를 유지한다.
- 네이버는 본문을 스니펫으로 추출하는 경우가 많으므로, 허브 본문 첫 문장 품질을 메타와 같은 톤/키워드로 관리한다.
- `title`/`description`/`og:title`/`og:description`은 “같은 메시지, 채널별 적정 길이” 원칙으로 맞춘다.
- 허브 메타에는 상세 숫자열(대출원금/월상환/총이자 등)을 넣지 않는다.

### 적용 기준 (일관성 체크)
- 허브 메타와 허브 본문 상단 문장이 의미적으로 일치한다.
- 금융 계산 블록 문구는 상세 하단에만 있고, 허브/상세 상단 메타 톤을 오염시키지 않는다.
- 배포 전 소스 기준 확인 파일: `app/page.tsx`, `app/layout.tsx`.
- 배포 후 3~7일 동안 네이버/구글 스니펫에서 숫자열 우선 노출 여부를 모니터링한다.
