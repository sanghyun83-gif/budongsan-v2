# MOBILE_UX_VISUAL_REFACTOR_IMPLEMENTATION_PLAN_2026-04-27

## 1) 배경 / 문제 정의
현재 모바일에서 다음 문제가 확인됨.

1. **다크모드 색상 충돌**
   - `app/globals.css`의 `prefers-color-scheme: dark`가 전역 변수는 다크로 바꾸지만,
   - 컴포넌트 스타일은 라이트 고정(`background: #fff`, `color: #0f172a`)이 많아 화면 톤이 깨짐.

2. **모바일 가독성 저하**
   - 11~13px 텍스트 사용 비중이 높아 본문/보조텍스트/버튼 라벨 가독성 저하.

3. **헤더/검색 영역 모바일 밀도 과다**
   - 데스크탑 레이아웃을 축소한 형태라 모바일에서 여백/계층/클릭 동선이 답답함.

4. **브레이크포인트 정교화 부족**
   - `900px`, `560px` 중심의 거친 분기로 인해 360~430px 구간 최적화가 약함.

---

## 2) 목표 (Goals)

### G1. 모바일 친화 UI 복구
- 360/390/412/430px 기준에서 텍스트, 여백, 터치 영역이 자연스럽게 동작.

### G2. 색상 시스템 일관화
- 라이트/다크 중 하나를 정책적으로 고정하거나, 둘 다 지원 시 토큰 기반으로 완전 일치.

### G3. SEO/핵심 카피 가독성 개선
- 홈 상단 핵심 문구(제목/설명)가 모바일에서 읽히고 클릭 유도 가능한 수준으로 개선.

### G4. 회귀 최소화
- 검색 기능/API/라우팅/데이터 로직은 변경하지 않고 CSS/표현 계층 위주 수정.

## 5) 실행 전략

## Phase 0. 정책 확정 (0.5일)

### 0-1) 컬러 모드 정책 결정 (둘 중 하나)
- **Option A (권장, 빠름): 라이트 고정**
  - `prefers-color-scheme: dark` 전역 덮어쓰기 제거.
  - 이유: 현재 컴포넌트가 라이트 기반으로 설계되어 회귀 리스크 최소.
- Option B: 다크 정식 지원
  - 모든 고정 색상 제거 + 토큰화 필요(작업량 큼).

> 이번 스프린트는 Option A 채택 권장.

### 0-2) 모바일 타이포 기준선
- 본문 최소 14px
- 보조 최소 12~13px (11px 금지)
- CTA/버튼 최소 14px 또는 최소 높이 44px

---

## Phase 1. 디자인 토큰/기본 스타일 정리 (1일)

## 1-1) 전역 토큰 도입 (`:root`)
아래 토큰을 추가하고 하드코드 색상을 점진 치환.
- `--bg`, `--surface`, `--text-primary`, `--text-secondary`
- `--border-default`, `--brand-primary`, `--brand-primary-strong`
- `--radius-sm/md/lg`, `--space-2/3/4/5`
- `--font-body`, `--font-caption`

## 1-2) 다크모드 충돌 제거
- `@media (prefers-color-scheme: dark)` 블록 제거 또는 무력화.
- body/background/foreground를 라이트 토큰 고정.

## 1-3) 공통 컴포넌트 최소 가이드
- `.ui-button`, `.ui-input`, `.ui-card-link` 기준 크기 재정의
- 터치영역 최소 높이 44px 고정
- 포커스 링 대비 강화

---

## Phase 2. 헤더/검색 모바일 최적화 (1일)

## 2-1) 헤더 밀도 완화
- `.global-header-inner` 모바일에서 2열 최소화
- 메뉴 링크는 모바일에서 전부 노출하지 않고 핵심 1~2개만 유지 또는 접기 처리
- 로고/검색 행 간격 재조정

## 2-2) SEO 인트로 가독성 개선
- `.header-search-intro-title`: 13px → 16px
- `.header-search-intro-desc`: 11~12px → 14px
- 라인 높이 1.5 이상
- 모바일에서 최대 줄 길이 제한(가독성 향상)

## 2-3) 검색 입력/아이콘 정렬
- 아이콘 음수 마진 구조 개선(absolute 또는 padding-left 기반 정렬)
- 입력 높이 40px 이상 유지
- placeholder 대비(색상) 상향

---

## Phase 3. 콘텐츠 카드/리스트 모바일 규격화 (1일)

## 3-1) 카드/리스트 타이포 상향
- 결과 카드 제목: 16px
- 메타텍스트: 13~14px
- 금액/핵심 값: 15~16px

## 3-2) 간격/여백 재조정
- 모바일 기본 패딩 12~16px
- 카드 내부 요소 간 gap 6→8 또는 10
- 과도한 압축 제거

## 3-3) 상세 페이지 하단 CTA
- `.detail-cta-grid` 모바일에서 버튼 간격 확대
- 스크롤 시 시인성 유지(배경/보더 대비 강화)

---

## Phase 4. 브레이크포인트 재설계 (0.5일)

권장 구간:
- `<= 430px` (소형 모바일)
- `431~767px` (일반 모바일/소형 태블릿)
- `768~1023px` (태블릿)
- `>= 1024px` (데스크탑)

기존 `900px`, `560px` 중심 규칙을 위 구간으로 정리.

## 9) 리스크 및 대응

1. **다크모드 제거 반발**
- 대응: 릴리즈 노트에 “현재 라이트 고정 안정화” 명시, 추후 다크모드 재도입 계획 별도 수립.

2. **CSS 영향 범위 과다**

3. **카피 줄수 증가로 레이아웃 붕괴**
- 대응: 모바일에서 제목/설명 max-width와 line-height 고정.
